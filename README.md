# TacWave

**TacWave** is a tactical radio log reader for **Command: Modern Operations**.

TacWave monitors the game log file in real time, detects relevant tactical events, groups repeated alerts, prioritizes threats, and reads them aloud using a simulated radio transmission voice.

It is designed to behave more like a **CIC / command center radio operator** than a simple text-to-speech tool.

The main goal is to reduce audio overload during complex scenarios by grouping related events and transmitting them as clear tactical radio messages.

---

## Version

**TacWave v1.0.0**

First public release.

---

## Important: start the scenario first

Before selecting a log file in TacWave, you should first start or load a scenario in **Command: Modern Operations**.

TacWave does not create the log file. It only reads an existing log file generated by the game.

Recommended startup order:

1. Open **Command: Modern Operations**.
2. Start or load the scenario.
3. Let the scenario run for a few seconds so the initial log file is created.
4. Open **TacWave**.
5. Select the active log file generated by the game.
6. Press **Start listening**.

If you select an old log file, or a file that is not currently being updated by the running scenario, TacWave may open normally but no new radio calls will be generated.

## What TacWave does

TacWave reads the Command: Modern Operations log and converts important events into spoken radio-style messages.

Instead of reading every log line one by one, TacWave tries to summarize the tactical picture.

Example of the intended behavior:

```text
CIC. Multiple incoming missiles from the north. Eight contacts. Hostile air formation to the northwest. Kidd impacted. Flooding. Over.
```

Instead of many repeated individual calls like:

```text
Missile incoming.
Missile incoming.
Missile incoming.
New air contact.
New air contact.
New air contact.
Damage report.
Damage report.
Damage report.
```

TacWave is especially useful in busy scenarios where many contacts, weapons, damage messages or classification updates are generated in a short period of time.

# Configuration options

This section explains every main option in TacWave and what each parameter does.

---

## Log file

The **Log file** field selects the Command: Modern Operations log file that TacWave will monitor.

This should be the active log file currently being updated by the running scenario.

Important points:

- TacWave does not create logs.
- Command: Modern Operations must already be running or the scenario must already be loaded.
- The scenario should run for a few seconds before selecting the log file.
- If you select an old log file, TacWave will not receive new events.
- If TacWave opens but does not speak, the most common cause is selecting the wrong log file.

Recommended workflow:

```text
1. Start the scenario in Command: Modern Operations.
2. Wait a few seconds.
3. Open TacWave.
4. Select the newest active log file.
5. Press Start listening.
```

---

## Voice mode

The **Voice mode** option controls how much information TacWave will speak.

It acts as a priority filter.

---

### Silent

**Silent** mode only speaks high-priority alerts.

Use this mode when you only want critical tactical warnings.

Recommended for:

- Very busy scenarios.
- Missile-heavy engagements.
- Large air/naval battles.
- Situations where you only want urgent threats.
- Reducing radio traffic as much as possible.

Typical alerts spoken in Silent mode:

- Incoming missiles.
- Severe damage.
- Destruction.
- Sinking.
- Critical impacts.

This mode is useful when the tactical environment is saturated and you do not want TacWave to report every lower-priority event.

---

### Tactical

**Tactical** mode is the recommended default mode.

It provides a balanced amount of information.

TacWave will speak important tactical events while filtering out lower-priority noise.

Recommended for:

- Normal gameplay.
- Medium to large scenarios.
- General tactical awareness.
- Balanced radio traffic.

This is the best mode for most users.

---

### Detailed

**Detailed** mode speaks more events.

It is useful for testing, debugging or slower scenarios where you want more information from the log.

Recommended for:

- Small scenarios.
- Testing TacWave.
- Checking if the parser is detecting events.
- Situations where you want more detailed reporting.

Not recommended for very large or busy scenarios, because it can become too verbose.

---

## System voice

The **System voice** option selects the installed Windows voice used by TacWave.

The available voices depend on:

- Your Windows language.
- Installed Windows speech packs.
- Installed text-to-speech voices.

Some voices sound more natural than others when processed through the radio effect.

If the voice sounds unclear, try another installed voice.

---

## Audio style

The **Audio style** option controls the audio processing applied to the voice.

---

### Clean

**Clean** mode uses standard Windows text-to-speech output.

No radio effect is applied.

Recommended for:

- Maximum clarity.
- Testing.
- Users who do not want radio distortion.
- Debugging detected messages.

This is the clearest mode, but the least immersive.

---

### Tactical radio

**Tactical radio** is the recommended audio mode.

It applies radio-style processing to the voice, including:

- Narrow-band filtering.
- Compression.
- Mild distortion.
- Simulated static.
- Signal variation.
- PTT opening and closing effects.

Recommended for:

- Normal gameplay.
- Tactical immersion.
- Clear but radio-like communications.

This mode tries to keep speech understandable while still sounding like a radio transmission.

---

### Degraded radio

**Degraded radio** applies a stronger radio effect.

It adds more:

- Distortion.
- Static.
- Signal degradation.
- Harshness.
- Radio character.

Recommended for:

- Immersion.
- Cinematic gameplay.
- Harsh radio atmosphere.

Not recommended if you need maximum clarity, because speech may become harder to understand.

---

## PTT radio

The **PTT radio** option enables or disables the simulated push-to-talk effect.

When enabled, TacWave adds radio opening and closing sounds around each transmission.

This helps each message feel like an actual radio call.

Recommended value:

```text
Enabled
```

When disabled, TacWave will still speak, but without the radio opening and closing effect.

---

## Radio noise

The **Radio noise** option controls the amount of simulated radio static and signal noise.

Higher values increase the amount of:

- Hiss.
- Static.
- Radio texture.
- Signal degradation.

Lower values make the voice cleaner and easier to understand.

Recommended values:

```text
Tactical radio: 4 to 8
Degraded radio: 2 to 5
```

If the voice becomes hard to understand, reduce this value.

If you want more immersion, increase it.

Recommended default:

```text
8
```

---

## Volume

The **Volume** option controls the speech volume.

Recommended value:

```text
80
```

This gives a strong voice level without being too aggressive.

If TacWave is too loud compared to the game, lower this value.

If TacWave is hard to hear, increase it.

---

## Voice speed

The **Voice speed** option controls how fast the voice speaks.

Recommended value:

```text
-1
```

A slightly slower voice usually sounds more natural for radio calls and is easier to understand during complex scenarios.

Suggested values:

```text
-2: slower and clearer
-1: recommended
 0: normal speed
 1 or higher: faster, but may reduce clarity
```

For busy scenarios, slower voice speed usually works better.

# Tactical grouping options

TacWave can group related events into a single radio transmission.

This prevents repeated alerts when many similar events happen at the same time.

Without grouping, a missile salvo or aircraft formation could generate many individual voice calls.

With grouping enabled, TacWave can say something like:

```text
CIC. Multiple incoming missiles from the north. Eight contacts. Closest, nine miles. Over.
```

instead of calling each missile one by one.

---

## Group missiles

The **Group missiles** option groups multiple missile detections into one warning.

Recommended value:

```text
Enabled
```

Use this to avoid repeated missile alerts when several missiles are detected almost at the same time.

Example:

```text
CIC. Multiple incoming missiles from the north. Eight contacts. Closest, nine miles. Over.
```

If this option is disabled, TacWave may speak each missile separately.

Recommended for all scenarios.

---

## Group aircraft

The **Group aircraft** option groups multiple aircraft contacts detected in the same sector.

Recommended value:

```text
Enabled
```

Use this when aircraft are detected in formations or groups.

Example:

```text
CIC. Hostile air formation to the northwest. Six contacts. Closest, thirty-three miles. Over.
```

This is useful when several aircraft are detected at once and should be treated as a formation rather than individual unrelated events.

---

## Group surface

The **Group surface** option groups multiple surface contacts.

This includes contacts such as:

- Ships.
- SKUNK contacts.
- Surface groups.
- Civilian surface traffic.

Recommended value:

```text
Enabled
```

Use this to avoid repeated surface contact reports when several ships are detected in the same area.

---

## Group hostility

The **Group hostility** option groups several contacts being marked, considered or identified as hostile.

Recommended value:

```text
Enabled
```

This is useful when multiple contacts change hostility status around the same time.

Example:

```text
CIC. Multiple contacts declared hostile to the northwest. Over.
```

If disabled, TacWave may report each hostility change separately.

---

## Group damage

The **Group damage** option groups repeated damage messages affecting the same unit.

Recommended value:

```text
Enabled
```

This is very important because Command: Modern Operations may generate many damage lines in a short time for:

- Weapon impacts.
- Blast damage.
- Fire.
- Flooding.
- System damage.
- Destruction.
- Sinking.

Instead of saying every damage line separately, TacWave tries to summarize the damage state.

Example:

```text
CIC. Kidd impacted. Fire and flooding. Over.
```

This prevents the radio from becoming overloaded during heavy attacks.

---

## Group classification

The **Group classification** option groups repeated contact classification updates.

Recommended value:

```text
Enabled
```

This is useful when several contacts are classified around the same time.

For example, several unknown air contacts may be classified as fighters, multirole aircraft or guided weapons.

If disabled, TacWave may speak each classification update separately.

# Contact window and missile window

The **Contact window** and **Missile window** options control how long TacWave waits before speaking grouped alerts.

These options define the balance between:

```text
Fast alerts vs grouped alerts
```

A lower value means TacWave speaks faster, but may generate more separate transmissions.

A higher value means TacWave waits longer, but can group more events into fewer transmissions.

---

## Contact window

The **Contact window** defines how many seconds TacWave waits before speaking grouped contact alerts.

This window is used mainly for non-missile tactical events such as:

- Aircraft contacts.
- Surface contacts.
- Hostility changes.
- Classification updates.
- General contact grouping.

A higher value gives TacWave more time to collect several related detections and speak them as one transmission.

Example with a contact window of 5 seconds:

```text
Several aircraft are detected in the same sector within a few seconds.
TacWave waits briefly, groups them, and speaks:
"CIC. Hostile air formation to the northwest. Six contacts. Closest, thirty-three miles. Over."
```

If the value is too low:

- TacWave may speak contacts too quickly.
- Similar contacts may not be grouped.
- Radio traffic may increase.

If the value is too high:

- TacWave may wait too long before speaking non-critical information.
- The tactical picture may feel delayed.

Recommended value:

```text
5 seconds
```

For very busy scenarios:

```text
6 to 8 seconds
```

For faster, more immediate calls:

```text
2 to 4 seconds
```

---

## Missile window

The **Missile window** defines how many seconds TacWave waits before speaking grouped missile alerts.

Missile alerts are more time-critical than normal contacts, so this value should usually be shorter than the contact window.

A short missile window allows TacWave to group missiles fired almost at the same time without delaying the warning too much.

Example with a missile window of 3 seconds:

```text
Several missiles are detected almost together.
TacWave groups them and speaks:
"CIC. Multiple incoming missiles from the north. Eight contacts. Closest, nine miles. Over."
```

If the value is too low:

- Each missile may be spoken separately.
- Missile salvos may become repetitive.
- Radio traffic may increase.

If the value is too high:

- Missile warnings may be delayed too much.
- Time-critical alerts may arrive late.

Recommended value:

```text
3 seconds
```

For very busy missile scenarios:

```text
2 to 3 seconds
```

For very fast alerts:

```text
1 to 2 seconds
```

General rule:

```text
Higher window = fewer, more grouped transmissions.
Lower window = faster, more frequent transmissions.
```

# Radio operator behavior

TacWave does not speak every alert separately if several alerts are waiting.

Instead, it combines them into a single radio transmission when possible.

Example:

```text
CIC. Multiple incoming missiles from the north. Hostile air formation to the northwest. Kidd impacted. Over.
```

This is intended to sound more like a CIC or command center operator and reduce audio overload.

The idea is not to report every single log line, but to provide a useful tactical radio picture.

# Troubleshooting

## TacWave does not speak

Check the following:

- The selected log file is the active log being updated by Command: Modern Operations.
- Windows audio output is working.
- Volume is not set to zero.
- The selected Windows voice is available.
- Voice mode is not filtering out low-priority events.
- The scenario is generating relevant log messages.

---

## TacWave opens but nothing happens

This usually means the selected log file is not being updated.

Recommended fix:

```text
1. Start or load the scenario in Command: Modern Operations.
2. Let the scenario run for a few seconds.
3. Find the newest log file generated by the game.
4. Select that file in TacWave.
5. Press Start listening.
```

---

## Too many alerts

Use these settings:

```text
Voice mode: Tactical
Group missiles: Enabled
Group aircraft: Enabled
Group surface: Enabled
Group damage: Enabled
Group classification: Enabled
```

For very busy scenarios, use:

```text
Voice mode: Silent
```

You can also increase:

```text
Contact window: 6 to 8 seconds
```

This gives TacWave more time to group contacts into fewer radio transmissions.

---

## Missile alerts are too repetitive

Enable:

```text
Group missiles: Enabled
```

Then use:

```text
Missile window: 2 to 3 seconds
```

If missiles are still being called one by one, slightly increase the missile window.

---

## Radio audio is too noisy

Reduce the value:

```text
Radio noise
```

Recommended values:

```text
Tactical radio: 4 to 8
Degraded radio: 2 to 5
```

---

## Voice is hard to understand

Try:

```text
Audio style: Tactical radio
Radio noise: 4
Voice speed: -1
Volume: 80
```

If needed, switch to:

```text
Audio style: Clean
```

# Limitations

TacWave depends on the text format generated by Command: Modern Operations logs.

Some events may not be detected if:

- The log format changes.
- The scenario uses different message wording.
- The relevant event type is not being written to the log.
- The selected file is not the active log file.

Radio audio is simulated locally and may sound different depending on installed Windows voices and audio devices.
